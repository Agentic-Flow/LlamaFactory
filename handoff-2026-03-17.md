## Executive Recommendation

For a first practical fine-tuning run, use:

- Model: `Qwen/Qwen3.5-0.8B`
- Training method: `LoRA SFT`
- Template: `qwen3_5_nothink`

Do not start with `thinking` mode unless you have a specific reasoning dataset and a clear reason to preserve long chain-of-thought behavior.

## Why This Path

Based on the official Qwen materials and the current `LlamaFactory` support:

- `Qwen/Qwen3.5-0.8B` is the better starting point for ordinary assistant and instruction tuning.
- `Qwen/Qwen3.5-0.8B-Base` is more appropriate for continued pretraining or for rebuilding assistant behavior from a base model.
- The current `LlamaFactory` tree already supports Qwen3.5 natively, including model registration and dedicated templates, so there is no need to add model-side adaptation first.

## Template Choice

Use these rules:

- If you fine-tune `Qwen/Qwen3.5-0.8B`, use `template: qwen3_5_nothink` for a normal non-reasoning SFT run.
- If you intentionally train with reasoning traces and want the reasoning template, use `template: qwen3_5`.
- Keep training and inference templates exactly the same.

## First-Pass Training Config

Create a YAML similar to this:

```yaml
### model
model_name_or_path: Qwen/Qwen3.5-0.8B
trust_remote_code: true

### method
stage: sft
do_train: true
finetuning_type: lora
lora_rank: 8
lora_target: all

### dataset
dataset: my_qwen35_sft
template: qwen3_5_nothink
cutoff_len: 2048
preprocessing_num_workers: 8
dataloader_num_workers: 4

### output
output_dir: saves/qwen3.5-0.8b/lora/sft
logging_steps: 10
save_steps: 500
plot_loss: true
overwrite_output_dir: true
save_only_model: false
report_to: none

### train
per_device_train_batch_size: 1
gradient_accumulation_steps: 8
learning_rate: 1.0e-4
num_train_epochs: 3.0
lr_scheduler_type: cosine
warmup_ratio: 0.1
bf16: true

### eval
# val_size: 0.1
# per_device_eval_batch_size: 1
# eval_strategy: steps
# eval_steps: 500
```

## Low-VRAM Variant

If GPU memory is tight, switch to 4-bit QLoRA:

```yaml
quantization_bit: 4
quantization_method: bnb
```

This follows the existing quantized example pattern already present in the repo.

## Dataset Format

The easiest path in this repository is Alpaca-style SFT data.

Example:

```json
[
  {
    "instruction": "把下面句子翻译成英文",
    "input": "今天天气很好。",
    "output": "The weather is very nice today."
  }
]
```

Register it in `data/dataset_info.json`:

```json
"my_qwen35_sft": {
  "file_name": "my_qwen35_sft.json"
}
```

Then point the YAML to:

```yaml
dataset: my_qwen35_sft
```

## Important Caveats

- Do not casually mix `qwen3_5` and `qwen3_5_nothink`.
- Use the same template during training and inference.
- If the dataset does not contain reasoning traces, do not force a reasoning-oriented setup as the first experiment.
- If the goal is domain adaptation of raw text rather than assistant behavior, consider `Qwen/Qwen3.5-0.8B-Base` and a continued pretraining style workflow instead of plain SFT.

## Practical Starting Point

If the goal is "get a usable domain assistant working quickly", the safest first run is:

1. Prepare a small clean Alpaca-style dataset.
2. Fine-tune `Qwen/Qwen3.5-0.8B` with LoRA.
3. Use `template: qwen3_5_nothink`.
4. Evaluate the LoRA adapter before trying thinking mode, DPO, or larger-scale data.
